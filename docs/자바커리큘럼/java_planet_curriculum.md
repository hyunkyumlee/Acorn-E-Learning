# Java 행성형 커리큘럼 설계안

## 1. 운영 구조

- 단계는 **Bronze → Silver → Gold** 순서이며, 단계마다 행성 5개로 구성한다.
- 행성 하나는 **이론 10개**로 구성한다.
- 이론 하나를 완료하면 해당 이론의 **일반문제 10개**를 푼다.
- 일반문제는 객관식, 빈칸 채우기, 코드 결과 예측, 간단 코드 입력을 학습 주제에 맞게 섞는다.
- 이론별 문제에서 **7문제 이상 정답(70%)**이면 이론을 통과한다. 한 행성의 이론 10개를 모두 통과하면 다음 행성이 열린다.
- 단계의 5번째 행성을 완료하면 다음 단계의 1번째 행성이 열린다.

## 난이도(B/S/G)별 일반문제 10개 구성 — 확정 (2026-07-23)

노션 비즈니스 산출물의 슬롯 표(손코딩 10번 등) 대신 **아래 규칙을 기준**으로 한다.

| 난이도 | 이론 (객/주) | 간단한 코딩 (객/주) | 합계 |
|--------|-------------|-------------------|------|
| Bronze | 4 | 6 | 10 |
| Silver | 2 | 8 | 10 |
| Gold | 객관 1 | 9 | 10 |

### 출제 순서

- 세트 **앞쪽은 객관식**, **뒤쪽은 주관식**.
- 이론·코딩 문항이 **섞여** 나올 수 있다 (이론 블록 → 코딩 블록 순서 고정 아님).

### 객관 / 주관 비율

| 구분 | 목표 비율 | 비고 |
|------|-----------|------|
| 이론 (객+주 합) | 객관 : 주관 = **1 : 1** | 나누어떨어지지 않으면 유동 분배 |
| 간단 코딩 (객+주 합) | 객관 : 주관 = **1 : 2** | 나누어떨어지지 않으면 유동 분배 |
| Gold 이론 1문항 | **객관식만** | — |

### 코딩 문항 유형

- 빈칸 채우기, 코딩 결과 예측, 코딩 한 줄 작성 등.
- 각 유형은 **객관식 또는 주관식**으로 출제 가능.
